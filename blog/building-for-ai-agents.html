Building docs and APIs for AI agents, not just humans

By Akshaya Chandramouli, Developer Relations — December 19, 2025

Roughly 40% of the reads on our developer docs now come from AI agents and code copilots, not humans. That's both a forcing function and a feature: the agents are the same ones helping our customers integrate Evolve, so making docs they can navigate is making developers happier downstream. We started by publishing llms.txt and llms-full.txt on the developer site so an agent landing on the docs has a structured map of the content; we then exposed Evolve itself as an MCP server, so agents can call the API directly during a session rather than re-implementing what the SDK already does.

The biggest thing we changed wasn't infrastructure — it was content shape. Headings as questions. Short, declarative answers. Code examples copy-pasteable without modification. No "as you can see above"-style references that break when an excerpt gets pulled out of context. We're still figuring out the right pattern for richer formats (diagrams, interactive examples) but the basics get us a long way.
